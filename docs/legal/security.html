Security Policy

Last updated: February 2025

Our Security Posture

The Tech Brief is a static website hosted on GitHub Pages. As a static site with no server-side execution, no user account system, and no database, our attack surface is minimal by design. We do not store passwords, payment information, or sensitive personal data.

HTTPS

This Site is served exclusively over HTTPS. All connections are encrypted using TLS, enforced by GitHub Pages' infrastructure. Unencrypted HTTP requests are automatically redirected to HTTPS.

Third-Party Services

Where we use third-party services (such as Formspree for contact forms or Google AdSense for advertising), we select providers with established security practices and verify their compliance where possible. We do not share sensitive personal data with third parties beyond what is necessary to operate the Site.

Responsible Disclosure

If you discover a security vulnerability on this Site or in any service we operate, we ask that you report it to us privately before disclosing it publicly. To report a vulnerability, email security@thetechbrief.in with:

  • A description of the vulnerability and its potential impact.
  • Steps to reproduce the issue.
  • Any relevant screenshots or proof-of-concept code.

We aim to acknowledge all valid security reports within 72 hours and will work to resolve confirmed issues as quickly as possible. We appreciate responsible disclosure and will credit reporters where desired.

No Bug Bounty Programme

We do not currently operate a paid bug bounty programme. However, we are grateful for responsible disclosures and will acknowledge valid reports publicly if requested by the reporter.

Contact

For security-related matters: security@thetechbrief.in
